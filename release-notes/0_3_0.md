# Release 0.3.0 — The Workspace Substrate

The framework now ships an event-sourced, replay-driven workspace substrate.
Where 0.1.0 introduced `Widget<P, W>` as the unit of composition and 0.2.0
landed the workspace primitives (SplitPane, MultiTabPane, Modal, drag), 0.3.0
turns the workspace itself into a substrate: a reusable shell that records
every structural mutation as a typed event, folds those events into an
in-memory state model, and projects the model onto the MultiTabPane in one
go. Refresh becomes a non-event. Theme change becomes a non-event. The page
just comes back the way you left it.

## What's in the box

### `homing-workspace-shell` — V2 workspace substrate (new module)

The reusable orchestrator behind any workspace app. Replaces the monolithic
chrome (V1) with constructor-injected substrate parts, each with a single
responsibility:

| Part | Role |
|------|------|
| `WorkspaceStateModel` | Pure state container. Layout tree + tabsBySlot + activeUuid + split ratios. Folds events; serialises to a checkpoint snapshot. |
| `ReplayEngine` | Fold-style reducer: `(state, event) -> state`. No DOM, no handler dispatch, no halt-on-error context — pure data. |
| `CheckpointService` | Cadence-driven snapshots. Worker-thread path via `CheckpointWorkerModule` (off the main thread), main-thread fallback when the worker is unavailable. |
| `EventEmitter` + `WriteLockGuard` | Web-Locks-backed single-writer fence; `pagehide`/`beforeunload` release prevents same-tab-reload lock contention. |
| `WorkspaceDirectory` | Multi-instance workspace identity (UUID + name). |
| `WorkspaceControlModal` | Identity / switcher / reset, themed with typed `StudioVars` JS color constants. |
| `WorkspaceShellChromeModule` | The orchestrator that wires the above and projects the model onto a MultiTabPane. |

### Virtual replay

Earlier prototypes replayed each event as a DOM mutation. 0.3.0 folds events
into the state model and projects once. A widget spawned then closed costs
zero DOM work on replay. The checkpoint snapshot IS the model serialised.

Event vocabulary now spans the full mutation surface:

- `WidgetSpawnedPinned` / `WidgetSpawnedFromPicker`
- `TabMoved` (within-pane reorder + cross-pane attach via `onTabAttached`)
- `TabClosed`
- `SplitCreated` / `SplitMerged`
- `SplitRatioChanged` (fired once on drag-stop with full layout; diff finds changed splits)
- `WorkspaceActiveChanged`

### `homing-workspace-codecs` — typed payload codecs (new module)

Foundation for typed event/checkpoint payloads. `WidgetParamsCodecRegistry`
refactored per the Substrate-Free Contracts doctrine (RFC 0035 P3).

### `homing-workspace-persistence` — IndexedDB event log + checkpoint store (new module)

`WorkspaceEventLogModule` and `CheckpointStoreModule` provide the storage
layer. Event pruning is opt-in via `window.__homing_enableEventPruning__`
— aggressive pruning is a footgun for replay correctness; off by default.

### `homing-codec` + `homing-codec-ecma` — codec generation foundation (new modules)

RFC 0034 P2 prep landing — JS codec generators driven by typed Java definitions.

### `homing-tree-views` — tree rendering substrate (new module)

`TreeNodeJsonWriter` + `TreeRendererModule` in `homing-core-js`; studio-base adapters.

### HTML Export — chrome include/exclude

`exportPageAsHtml(filename, opts)` now accepts `opts.includeChrome` (default
`true`). When `false`, elements marked `data-export-chrome` are stripped and
remaining `data-export-content` elements are lifted out of any layout
wrappers into a fresh centred `<main>` — producing a standalone-looking
document. DocReader and ComposedWidget both expose a "Include page chrome"
checkbox on the floating export pill.

## Breaking changes

None at the framework API level. V1 workspace chrome (`AnimalsPlaygroundChrome`)
remains alongside V2 as a parallel reference during the transition. No public
EsModule contract changed.

## Known limitations carried forward

- Floating-modal drag-detach state is not journaled (drag controller has no
  public callbacks for in-flight state).
- Cross-language replay needs typed payload codecs — partial (`homing-workspace-codecs`
  scaffolding present; full coverage tracked under RFC 0034 P2).
- Slow-motion replay deferred to a separate read-only app.
